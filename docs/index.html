Skill Profile

What this skill does

Guides AI agents through the full ClawStock crypto portfolio lifecycle — wallet-signature login, Arbitrum USDC Pool deposits, strategy discovery/invest/redeem/close, PnL tracking, position monitoring, and MAIN fund withdrawals. 18 structured commands mapped to REST API endpoints.

When to use it

User wants an AI agent to operate their ClawStock account — check balances, deposit USDC via the Pool contract, allocate funds to strategies, monitor positions and P&L, redeem shares, close strategies, or withdraw idle MAIN balance to their wallet.

What you get

Structured API integration with built-in safety guardrails — amount confirmation before every mutating operation, exchange/venue name suppression, no-private-key-by-design authentication flow, and clear MAIN↔STRATEGY account model semantics.

Who This Skill Is

ClawStock is designed for quantitative traders and crypto portfolio managers who delegate execution to AI agents. Wallet-signature login means the agent never touches private keys — custody stays with the user. The skill enforces confirmation gates on every deposit, invest, redeem, close, and withdraw action.

Best For

Crypto-native users running systematic strategies who want agent-driven portfolio operations with zero custody risk. Arbitrum USDC deposits, multi-strategy allocation, position-aware P&L tracking, and MAIN fund withdrawals — all via a single structured skill.

Account Model

MAIN ↔ STRATEGY Fund Flow

MAIN Account

User's primary account. Pool deposits credit MAIN. New strategy investments debit MAIN. After strategy close/redeem settlement, funds return to MAIN. Idle MAIN balance can be withdrawn to the user's wallet address.

STRATEGY Account

Isolated per-strategy account. Starting a strategy moves allocated amount from MAIN into this account. Closing or redeeming settles strategy funds according to backend status. Each strategy instance gets its own account.

Auth Flow

Wallet-Signature Login — 6 Steps

1
Collect wallet addressAsk user for their wallet address. Never ask for private keys, seed phrases, or wallet passwords.
2
Create challengePOST /api/v1/auth/challenge — send wallet_address, receive challenge_id + sign_url + result_token.
3
User signsGive the sign_url to the user. They open it in browser or mobile wallet — the page contains both signing UI and QR code.
4
Poll for resultGET /api/v1/auth/result/{challenge_id}?result_token=... — poll until verified: true.
5
Save API keyExtract verify.api_key from the result. Store in agent secret/session storage. Do not print it.
6
Authenticate requestsAll subsequent API calls use Authorization: Bearer <api_key>. The backend derives owner_address from the key — agent never touches it.

Safety

Safety Rules — 10 Guardrails

No private keys. Ever.

Wallet-signature login means the agent never touches seed phrases, private keys, wallet passwords, or recovery data. Custody identity (owner_address) is derived by the backend — the agent does not receive or transmit it.

Confirmation gates on every mutation.

Before /deposit, /submit-deposit, /invest, /redeem, /close, or /withdraw, the agent must confirm amount, asset, and strategy ID with the user. No silent execution.

Exchange/venue suppression.

API responses may include exchange fields or venue names. The agent must strip these from user-facing output. Report symbols and market types only.

No profit guarantees.

Never claim profit is guaranteed. Never describe products as principal-protected. Never provide personalized financial advice beyond the user's explicit ClawStock request.

Treat API keys as secrets.

Store only in agent secret/session storage. If a key is exposed in chat, instruct the user to rotate it when rotation is available.

MAIN withdrawals = user's own address only.

POST /api/v1/withdrawals sends funds to the authenticated owner_address. Do not ask for or accept a different destination. To exit a strategy, close/redeem it first, wait for settlement, then withdraw from MAIN.

Arbitrum USDC only for deposits.

Pool deposits currently support only USDC on Arbitrum. If the user requests another asset or network, explain the limitation clearly.
